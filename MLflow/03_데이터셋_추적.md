# 데이터셋 추적 (Dataset Tracking)

> 실험 추적을 먼저 익혔다면 → `02_실험_추적.md` 복습 후 진행

## 이번 실습 목표

- `mlflow.log_input()`으로 어떤 데이터로 학습했는지 기록한다.
- `mlflow.log_artifact()`로 파일 자체를 MLflow에 업로드한다.
- digest(데이터 고유 해시)로 데이터 버전을 구분한다.
- 기록된 데이터셋을 코드로 조회하고 다시 불러온다.

---

## 왜 데이터도 기록해야 하나?

파라미터와 메트릭만 기록하면 이런 문제가 생긴다.

```
Run 1: accuracy=0.85  ← 데이터 v1로 학습
Run 2: accuracy=0.90  ← 데이터 v2로 학습 (노이즈 제거함)
Run 3: accuracy=0.88  ← 데이터 v1로 학습 (실수로 되돌아감)
```

나중에 보면 Run 2가 왜 좋았는지 알 수 없다.
데이터셋도 함께 기록하면 "Run 2는 노이즈를 제거한 데이터를 썼기 때문"이라는 걸 추적할 수 있다.

---

## Dataset 객체란?

MLflow에서 데이터를 추적하는 공식 단위다.

```
Dataset 객체
├── name    : 데이터셋 이름     예) "my-dataset-v1"
├── digest  : 자동 계산 해시값  예) "4a8b2c1d..."  ← 데이터가 바뀌면 이 값도 바뀜
├── source  : 원본 위치         예) "data/train.csv"
├── schema  : 컬럼 타입 정보    (선택)
└── profile : 행 수, 통계 요약  (선택)
```

> **digest란?**
> 데이터 내용을 기반으로 자동 계산되는 고유 식별값이다.
> 데이터가 1개 행이라도 바뀌면 digest가 완전히 달라진다.
> "이 digest라면 이 데이터다" 라고 확신할 수 있다.

---

## log_input vs log_artifact

| | `log_input()` | `log_artifact()` |
|---|---|---|
| 저장 내용 | 이름·digest·경로 등 메타데이터 | 파일 자체 |
| UI 위치 | Inputs 탭 | Artifacts 탭 |
| 파일 삭제 시 | source 경로가 없으면 복원 불가 | MLflow 서버에 있으므로 복원 가능 |
| 용도 | 어떤 데이터를 썼는지 추적 | 파일 자체를 보관 |

## STEP 1. 데이터셋 메타데이터 기록 — `01_log_dataset.py`

`log_input()`으로 어떤 데이터를 썼는지 Run에 기록한다.
데이터 파일은 스크립트가 실행될 때 자동으로 생성된다.

```python
# 01_log_dataset.py
import os
import mlflow
import mlflow.data
import pandas as pd

mlflow.set_tracking_uri("sqlite:///C:/Users/myesu/Project/MLOps_Study/MLflow/practice/mlflow.db")

# v1: 5행, v2: 6행 — 버전 차이를 만들기 위해 데이터를 다르게 생성
os.makedirs("data", exist_ok=True)
with open("data/train_v1.csv", "w") as f:
    f.write("feature1,feature2,label\n1.0,2.0,0\n3.0,4.0,1\n5.0,6.0,0\n7.0,8.0,1\n9.0,10.0,0\n")
with open("data/train_v2.csv", "w") as f:
    f.write("feature1,feature2,label\n1.0,2.0,0\n3.0,4.0,1\n5.0,6.0,0\n7.0,8.0,1\n9.0,10.0,0\n11.0,12.0,1\n")

mlflow.set_experiment("dataset-tracking-demo")

def run_with_dataset(csv_path, dataset_name, run_name):
    df = pd.read_csv(csv_path)
    dataset = mlflow.data.from_pandas(
        df, source=csv_path, name=dataset_name, targets="label"
    )
    with mlflow.start_run(run_name=run_name):
        mlflow.log_input(dataset, context="training")  # ✅ 핵심
        mlflow.log_param("data_version", dataset_name)
        mlflow.log_metric("row_count", len(df))
        print(f"[{run_name}] name={dataset_name}, digest={dataset.digest[:8]}..., rows={len(df)}")

run_with_dataset("data/train_v1.csv", "my-dataset-v1", "v1-데이터-실험")
run_with_dataset("data/train_v2.csv", "my-dataset-v2", "v2-데이터-실험")
print("완료!")
```

```powershell

# [v2-데이터-실험] name=my-dataset-v2, digest=abaebf24..., rows=6
# 완료!
```

> v1과 v2의 digest가 다르다 — 데이터가 다르니까 당연하다.

## STEP 2. 파일 자체 업로드 — `02_log_artifact.py`

`log_artifact()`로 CSV 파일을 MLflow 서버에 직접 올린다.
UI **Artifacts 탭**에서 파일 내용을 직접 확인할 수 있다.

```python
# 02_log_artifact.py
import mlflow
import mlflow.data
import pandas as pd

mlflow.set_tracking_uri("sqlite:///C:/Users/myesu/Project/MLOps_Study/MLflow/practice/mlflow.db")
mlflow.set_experiment("dataset-tracking-demo")

with mlflow.start_run(run_name="artifact-업로드"):

    mlflow.log_artifact("data/train_v2.csv", artifact_path="datasets")

        df_v1, source="data/train_v1.csv", name="my-dataset-v1", targets="label"
    )
    mlflow.log_input(dataset_v1, context="training")

    print("완료! UI → artifact-업로드 Run → Artifacts 탭 확인")
```

## STEP 3. 분할 메타데이터 기록 — `03_log_splits.py`

train / val / test를 `context`로 구분해서 각각 기록한다.

```python
# 03_log_splits.py
import mlflow
import mlflow.data
import pandas as pd
from sklearn.model_selection import train_test_split

mlflow.set_tracking_uri("sqlite:///C:/Users/myesu/Project/MLOps_Study/MLflow/practice/mlflow.db")
mlflow.set_experiment("dataset-tracking-demo")

df = pd.read_csv("data/train_v2.csv")
train_df, temp_df = train_test_split(df, test_size=0.4, random_state=42)
val_df, test_df   = train_test_split(temp_df, test_size=0.5, random_state=42)

train_ds = mlflow.data.from_pandas(train_df, source="data/train_v2.csv", name="split-train")
val_ds   = mlflow.data.from_pandas(val_df,   source="data/train_v2.csv", name="split-val")
test_ds  = mlflow.data.from_pandas(test_df,  source="data/train_v2.csv", name="split-test")

with mlflow.start_run(run_name="split-tracking"):
    mlflow.log_input(train_ds, context="training")    # 학습 데이터
    mlflow.log_input(val_ds,   context="validation")  # 검증 데이터
    mlflow.log_input(test_ds,  context="testing")     # 테스트 데이터

    mlflow.log_metric("train_size", len(train_df))
    mlflow.log_metric("val_size",   len(val_df))
    mlflow.log_metric("test_size",  len(test_df))

    print(f"train={len(train_df)}, val={len(val_df)}, test={len(test_df)}")
    print("분할 추적 완료!")
```

## STEP 4. 분할 파일 업로드 — `04_log_splits_artifact.py`

분할된 데이터를 CSV로 저장하고 아티팩트로 올린다.
STEP 3(메타데이터) + STEP 2(파일 업로드)를 합친 버전이다.

```python
# 04_log_splits_artifact.py
import os
import mlflow
import mlflow.data
import pandas as pd
from sklearn.model_selection import train_test_split

mlflow.set_tracking_uri("sqlite:///C:/Users/myesu/Project/MLOps_Study/MLflow/practice/mlflow.db")
mlflow.set_experiment("dataset-tracking-demo")

df = pd.read_csv("data/train_v2.csv")
train_df, temp_df = train_test_split(df, test_size=0.4, random_state=42)
val_df, test_df   = train_test_split(temp_df, test_size=0.5, random_state=42)

# 분할된 DataFrame을 파일로 저장 (log_artifact는 파일을 올리므로)
os.makedirs("data/splits", exist_ok=True)
train_df.to_csv("data/splits/train.csv", index=False)
val_df.to_csv(  "data/splits/val.csv",   index=False)
test_df.to_csv( "data/splits/test.csv",  index=False)

train_ds = mlflow.data.from_pandas(train_df, source="data/train_v2.csv", name="split-train")
val_ds   = mlflow.data.from_pandas(val_df,   source="data/train_v2.csv", name="split-val")
test_ds  = mlflow.data.from_pandas(test_df,  source="data/train_v2.csv", name="split-test")

with mlflow.start_run(run_name="splits-with-artifact"):
    mlflow.log_input(train_ds, context="training")
    mlflow.log_input(val_ds,   context="validation")
    mlflow.log_input(test_ds,  context="testing")

    # UI Artifacts 탭 > splits/ 폴더에서 파일 확인 가능
    mlflow.log_artifact("data/splits/train.csv", artifact_path="splits")
    mlflow.log_artifact("data/splits/val.csv",   artifact_path="splits")
    mlflow.log_artifact("data/splits/test.csv",  artifact_path="splits")

    mlflow.log_metric("train_size", len(train_df))
    mlflow.log_metric("val_size",   len(val_df))
    mlflow.log_metric("test_size",  len(test_df))

    print(f"train={len(train_df)}, val={len(val_df)}, test={len(test_df)}")
    print("완료! UI → splits-with-artifact Run → Artifacts 탭 확인")
```

## STEP 6. 메타데이터 조회 — `05_get_metadata.py`

UI 없이 코드로 Run별 데이터셋 정보를 확인한다.

```python
# 05_get_metadata.py
import mlflow

mlflow.set_tracking_uri("sqlite:///C:/Users/myesu/Project/MLOps_Study/MLflow/practice/mlflow.db")

client = mlflow.tracking.MlflowClient()
experiment = client.get_experiment_by_name("dataset-tracking-demo")
runs = client.search_runs(experiment.experiment_id)

print("=== 실험별 사용 데이터셋 ===")
for run in runs:
    print(f"\nRun: {run.info.run_name}")
    print(f"  row_count : {run.data.metrics.get('row_count', 'N/A')}")
    for di in run.inputs.dataset_inputs:
        d = di.dataset
        print(f"  dataset  : {d.name}")
        print(f"  digest   : {d.digest}")
        # MLflow 3.x: di.tags는 딕셔너리가 아닌 InputTag 객체 리스트
        context = next((t.value for t in di.tags if t.key == "mlflow.data.context"), "N/A")
        print(f"  context  : {context}")
```

## STEP 7. 실제 데이터 조회 — `06_get_data.py`

메타데이터뿐 아니라 실제 행/열 데이터까지 불러온다.

```python
# 06_get_data.py
import mlflow
import mlflow.data
import pandas as pd

mlflow.set_tracking_uri("sqlite:///C:/Users/myesu/Project/MLOps_Study/MLflow/practice/mlflow.db")

client = mlflow.tracking.MlflowClient()
experiment = client.get_experiment_by_name("dataset-tracking-demo")
runs = client.search_runs(experiment.experiment_id)

for run in runs:
    print(f"\nRun: {run.info.run_name}")
    for di in run.inputs.dataset_inputs:
        d = di.dataset
        context = next((t.value for t in di.tags if t.key == "mlflow.data.context"), "N/A")
        print(f"  [{d.name}]  context={context}  digest={d.digest[:8]}...")
        try:
            source = mlflow.data.get_source(d)
            df = pd.read_csv(source.load())
            print(f"  행 수: {len(df)}  컬럼: {list(df.columns)}")
            print(df.to_string(index=False))
        except Exception as e:
            print(f"  데이터 로드 불가: {e}")
```

## STEP 8. source 경로로 복원 — `07_get_from_source.py`

파라미터로 특정 Run을 검색하고, 기록된 source 경로로 데이터를 가져온다.

```python
# 07_get_from_source.py
import mlflow
import mlflow.data
import pandas as pd

mlflow.set_tracking_uri("sqlite:///C:/Users/myesu/Project/MLOps_Study/MLflow/practice/mlflow.db")

client = mlflow.tracking.MlflowClient()
experiment = client.get_experiment_by_name("dataset-tracking-demo")

# data_version 파라미터로 v1 Run 검색 — Run ID 몰라도 된다
runs = client.search_runs(
    experiment.experiment_id,
    filter_string="params.data_version = 'my-dataset-v1'"
)

if not runs:
    print("v1 Run을 찾지 못했습니다. 01_log_dataset.py 를 먼저 실행하세요.")
    exit()

run = runs[0]
logged_dataset = run.inputs.dataset_inputs[0].dataset
print(f"name  : {logged_dataset.name}")
print(f"digest: {logged_dataset.digest}")

# source 경로의 파일을 그대로 읽어옴 — 원본 파일이 있어야 동작
source = mlflow.data.get_source(logged_dataset)
df = pd.read_csv(source.load())
print(f"\n복원된 데이터 ({len(df)}행):")
print(df)
```

> **주의:** 원본 파일(`data/train_v1.csv`)이 삭제되거나 경로가 바뀌면 실패한다.

## STEP 9. artifact에서 복원 — `08_get_from_artifact.py`

`log_artifact()`로 올린 파일을 MLflow 서버에서 다운로드해서 읽어온다.
원본 파일이 삭제되어도 복원할 수 있다.

```python
# 08_get_from_artifact.py
import os
import mlflow
import pandas as pd

mlflow.set_tracking_uri("sqlite:///C:/Users/myesu/Project/MLOps_Study/MLflow/practice/mlflow.db")

client = mlflow.tracking.MlflowClient()
experiment = client.get_experiment_by_name("dataset-tracking-demo")
runs = client.search_runs(experiment.experiment_id)

# 아티팩트가 있는 Run만 필터링
artifact_runs = [r for r in runs if client.list_artifacts(r.info.run_id)]

for run in artifact_runs:
    print(f"\nRun: {run.info.run_name}")
    download_dir = f"data/downloaded/{run.info.run_name}"
    os.makedirs(download_dir, exist_ok=True)

    for artifact in client.list_artifacts(run.info.run_id):
        local_path = client.download_artifacts(
            run_id=run.info.run_id,
            path=artifact.path,
            dst_path=download_dir
        )
        for root, dirs, files in os.walk(local_path):
            for file in files:
                if file.endswith(".csv"):
                    df = pd.read_csv(os.path.join(root, file))
                    print(f"  [{file}] {len(df)}행")
                    print(df.to_string(index=False))
```

## 핵심 API 정리

| 코드 | 설명 |
|------|------|
| `mlflow.data.from_pandas(df, source=, name=, targets=)` | Dataset 객체 생성 |
| `mlflow.log_input(dataset, context=)` | Run에 데이터셋 메타데이터 등록 |
| `mlflow.log_artifact(경로, artifact_path=)` | Run에 파일 자체 업로드 |
| `dataset.digest` | 데이터 고유 해시 확인 |
| `mlflow.data.get_source(dataset).load()` | source 경로의 파일 불러오기 |
| `client.download_artifacts(run_id, path, dst_path)` | 업로드된 artifact 다운로드 |

---

## 자주 하는 실수

| 실수 | 원인 | 해결 |
|------|------|------|
| UI Inputs 탭이 비어있음 | `log_artifact`만 쓰고 `log_input` 안 씀 | `mlflow.log_input(dataset, ...)` 사용 |
| v1과 v2 digest가 같음 | 파일 내용을 실제로 안 바꿈 | 데이터가 바뀌어야 digest가 달라짐 |
| `di.tags.get()` 오류 | MLflow 3.x에서 tags가 리스트 | `next((t.value for t in di.tags if t.key == "..."), "N/A")` 사용 |
| source로 복원 실패 | 원본 파일 경로가 없음 | `log_artifact()`로 파일 자체를 올려두면 해결 |
