# 데이터셋 추적 (Dataset Tracking)

> 실험 추적을 먼저 익혔다면 → `02_실험_추적.md` 복습 후 진행

## 이번 실습 목표

- `mlflow.log_input()`으로 어떤 데이터로 학습했는지 기록한다.
- digest(데이터 고유 해시)로 데이터 버전을 구분한다.
- 기록된 데이터셋을 나중에 다시 불러온다.

---

## 왜 데이터도 기록해야 하나?

파라미터와 메트릭만 기록하면 이런 문제가 생긴다.

```
Run 1: accuracy=0.85  ← 데이터 v1로 학습
Run 2: accuracy=0.90  ← 데이터 v2로 학습 (노이즈 제거함)
Run 3: accuracy=0.88  ← 데이터 v1로 학습 (실수로 되돌아감)
```

나중에 보면 Run 2가 왜 좋았는지 알 수 없다.
데이터셋도 함께 기록하면 "Run 2는 노이즈를 제거한 데이터를 썼기 때문"이라는 걸 추적할 수 있다.

---

## Dataset 객체란?

MLflow에서 데이터를 추적하는 공식 단위다.

```
Dataset 객체
├── name    : 데이터셋 이름     예) "iris-train-v2"
├── digest  : 자동 계산 해시값  예) "4a8b2c1d..."  ← 데이터가 바뀌면 이 값도 바뀜
├── source  : 원본 위치         예) "data/train.csv"
├── schema  : 컬럼 타입 정보    (선택)
└── profile : 행 수, 통계 요약  (선택)
```

> **digest란?**
> 데이터 내용을 기반으로 자동 계산되는 고유 식별값이다.
> 데이터가 1개 행이라도 바뀌면 digest가 완전히 달라진다.
> "이 digest라면 이 데이터다" 라고 확신할 수 있다.

## STEP 2. Dataset 객체 만들기

```python
# create_dataset.py
import mlflow
import mlflow.data
import pandas as pd

# 데이터 로드
df_v1 = pd.read_csv("data/train_v1.csv")
df_v2 = pd.read_csv("data/train_v2.csv")

# Dataset 객체 생성
dataset_v1 = mlflow.data.from_pandas(
    df_v1,
    source="data/train_v1.csv",   # 어디서 왔는지
    name="my-dataset-v1",          # 이름
    targets="label",               # 정답 컬럼
)

dataset_v2 = mlflow.data.from_pandas(
    df_v2,
    source="data/train_v2.csv",
    name="my-dataset-v2",
    targets="label",
)

# digest 확인
print(f"v1 digest: {dataset_v1.digest}")
print(f"v2 digest: {dataset_v2.digest}")
print(f"digest가 다른가? {dataset_v1.digest != dataset_v2.digest}")
```

```powershell
& $python create_dataset.py
# v1 digest: 004257c5
# v2 digest: abaebf24   ← 다름! 데이터가 바뀌었으니까
# digest가 다른가? True
```

> CSV 내용을 문서와 똑같이 입력했다면 digest 값도 위와 동일하게 나온다.
> digest는 데이터 내용으로 계산되므로, 같은 데이터면 항상 같은 값이다.

## STEP 3. Run에 데이터셋 등록하기

```python
# log_dataset.py
import mlflow
import mlflow.data
import mlflow.sklearn
import pandas as pd
from sklearn.linear_model import LogisticRegression
from sklearn.metrics import accuracy_score
from sklearn.model_selection import train_test_split

mlflow.set_experiment("dataset-tracking-demo")


def run_with_dataset(csv_path, dataset_name, run_name):
    df = pd.read_csv(csv_path)

    # Dataset 객체 생성
    dataset = mlflow.data.from_pandas(
        df,
        source=csv_path,
        name=dataset_name,
        targets="label",
    )

    with mlflow.start_run(run_name=run_name):
        # ✅ 데이터셋 등록 (핵심!)
        mlflow.log_input(dataset, context="training")

        # 학습
        X = df[["feature1", "feature2"]]
        y = df["label"]
        X_train, X_test, y_train, y_test = train_test_split(
            X, y, test_size=0.3, random_state=42
        )

        model = LogisticRegression()
        model.fit(X_train, y_train)

        acc = accuracy_score(y_test, model.predict(X_test))
        mlflow.log_metric("accuracy", round(acc, 4))

        print(f"[{run_name}] dataset={dataset_name}, digest={dataset.digest[:8]}..., accuracy={acc:.4f}")


run_with_dataset("data/train_v1.csv", "my-dataset-v1", "v1-데이터-실험")
run_with_dataset("data/train_v2.csv", "my-dataset-v2", "v2-데이터-실험")
```

```powershell
& $python log_dataset.py
# [v1-데이터-실험] dataset=my-dataset-v1, digest=004257c5..., accuracy=1.0000
# [v2-데이터-실험] dataset=my-dataset-v2, digest=abaebf24..., accuracy=1.0000
```

> accuracy가 1.0인 건 데이터가 6~8행으로 매우 작고 단순해서다 (의미 없음).
> 이 실습의 핵심은 **digest로 데이터 버전을 구분**하는 것이다 (v1 ≠ v2).

## STEP 5. 등록된 데이터셋 코드로 조회하기

```python
# get_dataset_info.py
import mlflow

client = mlflow.tracking.MlflowClient()
experiment = client.get_experiment_by_name("dataset-tracking-demo")

runs = client.search_runs(experiment.experiment_id)

print("=== 실험별 사용 데이터셋 ===")
for run in runs:
    print(f"\nRun: {run.info.run_name}")
    print(f"  accuracy : {run.data.metrics.get('accuracy', 'N/A')}")

    for di in run.inputs.dataset_inputs:
        d = di.dataset
        print(f"  dataset  : {d.name}")
        print(f"  digest   : {d.digest}")   # 데이터 버전 식별값
        # MLflow 3.x: di.tags는 딕셔너리가 아닌 InputTag 객체 리스트
        context = next((t.value for t in di.tags if t.key == "mlflow.data.context"), "N/A")
        print(f"  context  : {context}")
```

## STEP 6. 기록된 데이터셋 원본 복원하기

나중에 특정 Run에서 썼던 데이터를 다시 가져올 수 있다.

```python
# restore_dataset.py
import mlflow
import pandas as pd

# 복원할 Run ID (get_dataset_info.py 결과에서 확인)
RUN_ID = "..."   # 실제 Run ID로 교체

run = mlflow.get_run(RUN_ID)
logged_dataset = run.inputs.dataset_inputs[0].dataset

print(f"데이터셋 이름: {logged_dataset.name}")
print(f"digest     : {logged_dataset.digest}")

# 원본 파일로 복원
source = mlflow.data.get_source(logged_dataset)
local_path = source.load()   # 로컬에 다운로드

df = pd.read_csv(local_path)
print(f"\n복원된 데이터:")
print(df)
```

## STEP 7. 학습·검증·테스트 분할 각각 기록하기

하나의 Run에 여러 데이터셋을 context로 구분해서 등록할 수 있다.

```python
# log_splits.py
import mlflow
import mlflow.data
import pandas as pd
from sklearn.model_selection import train_test_split

mlflow.set_experiment("dataset-tracking-demo")

df = pd.read_csv("data/train_v2.csv")
train_df, temp_df = train_test_split(df, test_size=0.4, random_state=42)
val_df, test_df   = train_test_split(temp_df, test_size=0.5, random_state=42)

# 분할별 Dataset 객체
train_ds = mlflow.data.from_pandas(train_df, source="data/train_v2.csv", name="split-train")
val_ds   = mlflow.data.from_pandas(val_df,   source="data/train_v2.csv", name="split-val")
test_ds  = mlflow.data.from_pandas(test_df,  source="data/train_v2.csv", name="split-test")

with mlflow.start_run(run_name="분할-추적"):
    mlflow.log_input(train_ds, context="training")    # 학습 데이터
    mlflow.log_input(val_ds,   context="validation")  # 검증 데이터
    mlflow.log_input(test_ds,  context="testing")     # 테스트 데이터

    mlflow.log_metric("train_size", len(train_df))
    mlflow.log_metric("val_size",   len(val_df))
    mlflow.log_metric("test_size",  len(test_df))

    print("분할 데이터셋 등록 완료!")
```

## 지원하는 데이터 형식

| 함수 | 사용 대상 |
|------|----------|
| `mlflow.data.from_pandas(df, ...)` | Pandas DataFrame |
| `mlflow.data.from_numpy(array, ...)` | NumPy 배열 |
| `mlflow.data.from_spark(spark_df, ...)` | Spark DataFrame |

---

## 핵심 API 정리

| 코드 | 설명 |
|------|------|
| `mlflow.data.from_pandas(df, source=, name=, targets=)` | Dataset 객체 생성 |
| `mlflow.log_input(dataset, context="training")` | Run에 데이터셋 등록 |
| `dataset.digest` | 데이터 고유 해시 확인 |
| `mlflow.data.get_source(logged_dataset)` | 원본 소스 정보 가져오기 |
| `source.load()` | 원본 데이터 로컬로 복원 |

---

## 자주 하는 실수

| 실수 | 원인 | 해결 |
|------|------|------|
| UI Inputs 탭이 비어있음 | `log_artifact`만 쓰고 `log_input` 안 씀 | 반드시 `mlflow.log_input(dataset, ...)` 사용 |
| v1과 v2 digest가 같음 | 파일 내용을 실제로 안 바꿈 | 데이터가 바뀌어야 digest가 달라짐 |
| 복원 경로를 못 찾음 | `source` 경로가 다른 환경에선 없음 | 원격 저장소(S3 등) 사용 시 경로 주의 |
