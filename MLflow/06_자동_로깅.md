## 이번 실습 목표

- `mlflow.autolog()` 한 줄로 파라미터·메트릭·모델을 자동 기록한다.
- 수동 로깅(`log_param`, `log_metric`)과 자동 로깅의 차이를 이해한다.
- GridSearchCV와 autolog를 조합해 최적 파라미터를 자동으로 탐색하고 기록한다.

---

## autolog이란?

지금까지는 이렇게 하나씩 직접 기록했다.

```python
# 수동 로깅 (지금까지 방식)
mlflow.log_param("max_iter", 100)
mlflow.log_param("C", 1.0)
mlflow.log_metric("accuracy", 0.95)
mlflow.sklearn.log_model(model, "model")
```

`mlflow.autolog()`를 쓰면 이 모든 게 **자동**으로 기록된다.

```python
# 자동 로깅
mlflow.autolog()   # 이 한 줄이 전부!
```

> **비유:** 수동 로깅은 실험 노트를 직접 펜으로 쓰는 것,
> 자동 로깅은 보이스레코더가 실험 과정을 통째로 녹음하는 것이다.

## STEP 1. autolog 활성화 — `01_set_autolog.py`

```python
# 01_set_autolog.py
import mlflow
from sklearn.datasets import load_iris
from sklearn.linear_model import LogisticRegression
from sklearn.model_selection import train_test_split

mlflow.set_tracking_uri("sqlite:///C:/Users/myesu/Project/MLOps_Study/MLflow/practice/mlflow.db")
mlflow.set_experiment("autolog-test")

# ✅ start_run() 보다 반드시 먼저 호출해야 적용됨
mlflow.autolog()

iris = load_iris()
X_train, X_test, y_train, y_test = train_test_split(
    iris.data, iris.target, test_size=0.2, random_state=42
)

with mlflow.start_run(run_name="autolog-run"):
    model = LogisticRegression(max_iter=100, C=1.0)
    model.fit(X_train, y_train)   # fit() 호출 시점에 자동 기록됨
    print("autolog 실행 완료. UI에서 기록된 항목을 확인하세요.")
    print(f"Run ID: {mlflow.active_run().info.run_id}")
```

## STEP 2. 기록된 항목 확인 — `02_get_autolog.py`

autolog가 자동으로 기록한 항목을 코드로 조회한다.

```python
# 02_get_autolog.py
import mlflow

mlflow.set_tracking_uri("sqlite:///C:/Users/myesu/Project/MLOps_Study/MLflow/practice/mlflow.db")

client = mlflow.tracking.MlflowClient()

experiment = client.get_experiment_by_name("autolog-test")
runs = client.search_runs(experiment.experiment_id)

run = runs[0]
print(f"Status    : {run.info.status}")
print(f"Params    : {run.data.params}")
print(f"Metrics   : {run.data.metrics}")
print(f"Artifacts : {[f.path for f in client.list_artifacts(run.info.run_id)]}")
```

```powershell
& $python 02_get_autolog.py
# Status    : FINISHED
# Params    : {'C': '1.0', 'max_iter': '100', 'solver': 'lbfgs', ...}
# Metrics   : {'training_score': 0.9583}
# Artifacts : ['model']
```

### scikit-learn 기준 자동 기록 항목

| 종류 | 자동 기록 항목 |
|------|--------------|
| **Parameters** | `C`, `max_iter`, `solver`, `penalty` 등 모든 하이퍼파라미터 |
| **Metrics** | `training_score` (훈련 정확도) |
| **Artifacts** | `model/` 폴더 (MLmodel, model.pkl, conda.yaml 등) |
| **Tags** | `mlflow.autologging`, `mlflow.source.name`, `mlflow.user` |

> `training_score`만 자동 기록된다. test accuracy는 자동으로 기록되지 않으므로
> 필요하면 `mlflow.log_metric("test_accuracy", ...)` 수동으로 추가해야 한다.

## STEP 3. GridSearch + autolog — `03_autolog_gridsearch.py`

GridSearchCV의 핵심은 **가장 성능 좋은 파라미터 조합을 자동으로 찾는 것**이다.

```
C=0.1로 해보고, C=1.0으로 해보고... 하는 수동 작업을 GridSearch가 대신 해준다
      ↓
9가지 조합 전부 시도 → CV 점수 비교 → 최적 조합 자동 선택
      ↓
gs.best_params_  ← 가장 성능 좋은 조합
gs.best_score_   ← 그 조합의 점수
```

autolog는 여기에 **"어떤 조합이 얼마나 나왔는지"를 MLflow에 전부 기록**한다.

```python
# 03_autolog_gridsearch.py
import mlflow
from sklearn.datasets import load_iris
from sklearn.linear_model import LogisticRegression
from sklearn.model_selection import GridSearchCV, train_test_split

mlflow.set_tracking_uri("sqlite:///C:/Users/myesu/Project/MLOps_Study/MLflow/practice/mlflow.db")
mlflow.set_experiment("autolog-gridsearch")
mlflow.autolog()   # GridSearchCV의 9가지 조합을 자식 Run으로 자동 기록

iris = load_iris()
X_train, X_test, y_train, y_test = train_test_split(
    iris.data, iris.target, test_size=0.2, random_state=42
)

# C 3가지 × max_iter 3가지 = 총 9가지 조합
param_grid = {
    "C":        [0.1, 1.0, 10.0],
    "max_iter": [50, 100, 200],
}

with mlflow.start_run(run_name="grid-search"):
    gs = GridSearchCV(
        LogisticRegression(),
        param_grid,
        cv=3,           # 각 조합을 3-fold 교차 검증으로 평가 → 총 27번 학습
        scoring="accuracy",
    )
    gs.fit(X_train, y_train)   # fit() 시점에 9개 자식 Run 자동 생성
    print(f"최적 파라미터: {gs.best_params_}")
    print(f"최적 점수    : {gs.best_score_:.4f}")
```

```powershell
& $python 03_autolog_gridsearch.py
# 최적 파라미터: {'C': 1.0, 'max_iter': 50}
# 최적 점수    : 0.9583
```

### 부모-자식 Run 구조

```
부모 Run: "grid-search"
│  → best_C, best_max_iter, best_cv_score (최종 결과)
│
├── 자식 Run 1: C=0.1,  max_iter=50   → cv_score
├── 자식 Run 2: C=0.1,  max_iter=100  → cv_score
│   ...
└── 자식 Run 9: C=10.0, max_iter=200  → cv_score
```

UI에서 `autolog-gridsearch` 실험 → 부모 Run 왼쪽 `▶` 펼치면 자식 Run 9개가 나열된다.
`mean_test_score` 컬럼으로 정렬하면 어떤 조합이 가장 좋았는지 한눈에 비교 가능하다.

## STEP 4. 특정 항목만 기록 끄기

너무 많은 정보가 기록되어 정리가 안 될 때 일부만 끌 수 있다.

```python
mlflow.autolog(
    log_input_examples=False,   # 입력 예시 저장 안 함
    log_model_signatures=False, # 모델 시그니처 저장 안 함
    log_models=True,            # 모델 파일은 저장
    silent=True,                # 자동 로깅 관련 경고 메시지 숨김
)
```

## STEP 5. 프레임워크별 자동 로깅

`mlflow.autolog()`는 사용 중인 라이브러리를 자동 감지한다.
각 라이브러리마다 따로 활성화할 수도 있다.

```python
mlflow.sklearn.autolog()      # scikit-learn만
mlflow.pytorch.autolog()      # PyTorch만
mlflow.tensorflow.autolog()   # TensorFlow/Keras만
mlflow.xgboost.autolog()      # XGBoost만
```

### 프레임워크별 자동 기록 항목 비교

| 프레임워크 | 파라미터 | 메트릭 | 모델 | 추가 기록 |
|-----------|---------|--------|------|----------|
| **scikit-learn** | 전체 하이퍼파라미터 | training_score | ✅ | feature importance |
| **PyTorch** | - | 에포크별 loss | ✅ | 학습 곡선 |
| **TensorFlow** | - | 에포크별 loss, val_loss | ✅ | 학습 곡선 그래프 |
| **XGBoost** | 전체 하이퍼파라미터 | eval metric | ✅ | feature importance |

## 수동 로깅 vs 자동 로깅 — 언제 쓸까?

| 상황 | 추천 방법 |
|------|----------|
| 빠르게 실험 시작할 때 | `autolog()` |
| 기록할 항목을 직접 제어하고 싶을 때 | 수동 로깅 |
| 팀원과 기록 규칙을 통일할 때 | 수동 로깅 |
| GridSearchCV 등 최적 파라미터 탐색 | `autolog()` |
| 데이터셋도 함께 기록할 때 | 수동 + `log_input()` 조합 |

> **실무 팁:** `autolog()`로 시작하고, 자동으로 안 잡히는 것만 수동으로 추가하는 방식이 가장 편하다.

```python
# 추천 조합 패턴
mlflow.autolog()   # 기본은 자동으로

with mlflow.start_run():
    mlflow.log_input(dataset, context="training")         # 데이터셋 (자동 안 됨)
    mlflow.log_param("실험_목적", "노이즈제거-효과-검증") # 커스텀 메모
    model.fit(X_train, y_train)   # 나머지는 자동 기록
```

## 핵심 API 정리

| 코드 | 설명 |
|------|------|
| `mlflow.autolog()` | 모든 지원 프레임워크 자동 로깅 활성화 |
| `mlflow.sklearn.autolog()` | scikit-learn만 자동 로깅 |
| `mlflow.autolog(log_models=False)` | 모델 파일 저장 제외 |
| `mlflow.autolog(silent=True)` | 경고 메시지 숨김 |
| `gs.best_params_` | GridSearch 최적 파라미터 조합 |
| `gs.best_score_` | GridSearch 최적 조합의 CV 점수 |

---

## 자주 하는 실수

| 실수 | 원인 | 해결 |
|------|------|------|
| 자동 기록이 안 됨 | `autolog()` 를 `start_run()` 뒤에 씀 | `autolog()`는 반드시 `start_run()` **전에** 호출 |
| Run이 2개 생김 | autolog가 내부적으로 Run을 하나 더 만듦 | `with mlflow.start_run()` 으로 감싸면 하나로 합쳐짐 |
| test accuracy가 없음 | autolog는 `training_score`만 기록 | `mlflow.log_metric("test_accuracy", ...)` 수동 추가 |
| 너무 많은 Run | GridSearchCV 자식 Run이 수십 개 생김 | 정상 동작, UI 필터로 부모 Run만 보기 |
