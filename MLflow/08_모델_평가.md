## 이번 실습 목표

- `mlflow.evaluate()` 한 번으로 여러 평가 지표를 자동 계산한다.
- 혼동행렬·ROC 곡선 등 평가 그래프를 자동으로 기록한다.
- 여러 모델을 같은 기준으로 비교한다.

---

## 왜 evaluate인가?

지금까지는 정확도를 **직접 계산**해서 기록했다.

```python
# 지금까지 방식 (02~04장)
acc = accuracy_score(y_test, model.predict(X_test))
mlflow.log_metric("accuracy", acc)        # 지표 하나하나 직접
```

`mlflow.evaluate()`를 쓰면 이 모든 게 **자동**이다.

```python
mlflow.evaluate(model_uri, data=test_df, targets="target", model_type="classifier")
# → accuracy·precision·recall·f1·log_loss·roc_auc 자동 계산
# → 혼동행렬·ROC·PR 곡선 이미지까지 Run에 자동 저장
```

> **비유:** 직접 계산이 시험을 항목별로 일일이 채점하는 거라면,
> evaluate는 답안지를 넣으면 점수표·오답분석·그래프까지 한 번에 출력하는 자동 채점기다.

## STEP 1. 자동 평가 — `01_evaluate_basic.py`

```python
# 01_evaluate_basic.py
import mlflow, mlflow.sklearn, pandas as pd
from sklearn.datasets import load_iris
from sklearn.linear_model import LogisticRegression
from sklearn.model_selection import train_test_split

mlflow.set_tracking_uri("sqlite:///C:/Users/myesu/Project/MLOps_Study/MLflow/practice/mlflow.db")
mlflow.set_experiment("iris-evaluation")

# evaluate에 넘길 test 데이터는 정답(target) 컬럼을 포함해야 함
iris = load_iris()
df = pd.DataFrame(iris.data, columns=iris.feature_names)
df["target"] = iris.target
train_df, test_df = train_test_split(df, test_size=0.2, random_state=42)

X_train = train_df.drop("target", axis=1)
y_train = train_df["target"]

model = LogisticRegression(max_iter=200)
model.fit(X_train, y_train)

with mlflow.start_run(run_name="evaluate-basic"):
    # evaluate는 저장된 모델 URI를 받아 평가한다
    info = mlflow.sklearn.log_model(model, name="model")

    result = mlflow.evaluate(
        info.model_uri,
        data=test_df,
        targets="target",
        model_type="classifier",   # 분류 모델. 회귀면 "regressor"
    )

    print("=== 자동 계산된 평가 지표 ===")
    for key, value in result.metrics.items():
        print(f"  {key:20s}: {value:.4f}")
```

> iris는 쉬운 데이터라 지표가 전부 1.0으로 나온다. 실제 복잡한 데이터에서는
> 지표마다 값이 달라지며, 그 차이를 비교하는 게 평가의 핵심이다.

### 자동 생성되는 평가 지표 (classifier)

| 지표 | 의미 |
|------|------|
| `accuracy_score` | 전체 정확도 |
| `precision_score` | 정밀도 (예측이 맞을 확률) |
| `recall_score` | 재현율 (실제를 놓치지 않을 확률) |
| `f1_score` | 정밀도·재현율의 조화평균 |
| `log_loss` | 확률 예측의 손실 (낮을수록 좋음) |
| `roc_auc` | ROC 곡선 아래 면적 (1에 가까울수록 좋음) |

## STEP 3. 여러 모델 비교 — `02_compare_models.py`

```python
# 02_compare_models.py
import mlflow, mlflow.sklearn, pandas as pd
from sklearn.datasets import load_iris
from sklearn.linear_model import LogisticRegression
from sklearn.tree import DecisionTreeClassifier
from sklearn.model_selection import train_test_split

mlflow.set_tracking_uri("sqlite:///C:/Users/myesu/Project/MLOps_Study/MLflow/practice/mlflow.db")
mlflow.set_experiment("iris-evaluation")

iris = load_iris()
df = pd.DataFrame(iris.data, columns=iris.feature_names)
df["target"] = iris.target
train_df, test_df = train_test_split(df, test_size=0.2, random_state=42)

X_train = train_df.drop("target", axis=1)
y_train = train_df["target"]

# 비교할 모델 두 종류
models = {
    "logistic-regression": LogisticRegression(max_iter=200),
    "decision-tree":       DecisionTreeClassifier(max_depth=3, random_state=42),
}

results = {}
for name, model in models.items():
    model.fit(X_train, y_train)
    with mlflow.start_run(run_name=name):
        info = mlflow.sklearn.log_model(model, name="model")
        result = mlflow.evaluate(
            info.model_uri, data=test_df, targets="target", model_type="classifier"
        )
        results[name] = result.metrics
        print(f"[{name}] 평가 완료")

print("\n=== 모델 비교 ===")
print(f"{'지표':16s} {'logistic':>12s} {'decision-tree':>14s}")
for key in ["accuracy_score", "f1_score", "precision_score", "recall_score"]:
    lr = results["logistic-regression"].get(key, 0)
    dt = results["decision-tree"].get(key, 0)
    print(f"{key:16s} {lr:12.4f} {dt:14.4f}")
```

## STEP 4. 평가 결과 코드로 조회 — `03_get_eval_metrics.py`

```python
# 03_get_eval_metrics.py
import mlflow

mlflow.set_tracking_uri("sqlite:///C:/Users/myesu/Project/MLOps_Study/MLflow/practice/mlflow.db")

client = mlflow.tracking.MlflowClient()
experiment = client.get_experiment_by_name("iris-evaluation")

# accuracy_score 기준 내림차순 — 가장 잘 평가된 모델이 위로
runs = client.search_runs(
    experiment.experiment_id,
    order_by=["metrics.accuracy_score DESC"],
)

print("=== 평가 결과 순위 (accuracy 기준) ===")
for i, run in enumerate(runs, 1):
    name = run.info.run_name
    acc  = run.data.metrics.get("accuracy_score", 0)
    f1   = run.data.metrics.get("f1_score", 0)
    auc  = run.data.metrics.get("roc_auc", 0)
    print(f"{i}위 [{name}]  accuracy={acc:.4f}  f1={f1:.4f}  roc_auc={auc:.4f}")

    artifacts = [a.path for a in client.list_artifacts(run.info.run_id)]
    eval_files = [a for a in artifacts if a.endswith((".png", ".csv"))]
    print(f"     평가 아티팩트: {eval_files}")
```

## 핵심 API 정리

| 코드 | 설명 |
|------|------|
| `mlflow.evaluate(model_uri, data=, targets=, model_type=)` | 모델 자동 평가 |
| `model_type="classifier"` | 분류 모델 평가 (accuracy, f1, roc_auc ...) |
| `model_type="regressor"` | 회귀 모델 평가 (MAE, MSE, R² ...) |
| `result.metrics` | 계산된 지표 딕셔너리 |
| `result.artifacts` | 생성된 그래프·파일 목록 |

---

## 자주 하는 실수

| 실수 | 원인 | 해결 |
|------|------|------|
| `targets` 컬럼 없음 | data에 정답 컬럼을 안 넣음 | test 데이터에 target 컬럼 포함해서 전달 |
| 평가가 안 됨 | `model_type` 누락 | `"classifier"` 또는 `"regressor"` 지정 |
| 모델 URI 오류 | 저장 전에 evaluate 호출 | `log_model` 후 반환된 `info.model_uri` 사용 |
| 지표가 전부 1.0 | iris가 너무 쉬운 데이터 | 정상. 복잡한 데이터에서 차이가 드러남 |

## 마무리

이로써 MLflow의 핵심 흐름을 모두 실습했다.

```
실험 추적(02) → 데이터 추적(03) → 모델 관리(04)
   → 자동 로깅(06) → 모델 서빙(07) → 모델 평가(08)
```

학습/실험 → 기록 → 관리 → 배포 → 평가로 이어지는 ML 라이프사이클을
MLflow 하나로 다룰 수 있게 되었다.
