# MLflow란?

## MLflow가 뭔가요?

MLflow는 **머신러닝(ML) 실험을 기록하고 관리하는 오픈소스 도구**다.

쉽게 말하면 이런 상황을 해결해준다.

> "아까 그 모델이 정확도 95%였는데... 그때 학습률이 얼마였지?"
> "데이터를 이렇게 바꿨더니 성능이 떨어졌는데, 원래대로 어떻게 돌아가지?"
> "팀원이 만든 모델을 내가 쓰려면 어떻게 하지?"

MLflow를 쓰면 실험마다 **어떤 설정으로, 어떤 데이터로, 어떤 결과**가 나왔는지 자동으로 기록된다.

---

## 왜 필요한가요?

### ML 개발의 현실

```
처음: "학습률 0.01로 해보자" → 정확도 80%
두 번째: "학습률 0.001로 바꿔보자" → 정확도 85%
세 번째: "데이터 정제해보자" → 정확도 92%
네 번째: "레이어 추가해보자" → 정확도 78% (오히려 떨어짐)
다섯 번째: "아 세 번째로 돌아가야 하는데... 설정이 뭐였지?" 🤯
```

MLflow 없이 작업하면 생기는 문제:

| 문제 | 상황 |
|------|------|
| 재현 불가 | 잘 됐던 실험 설정을 잊어버림 |
| 비교 어려움 | 어떤 변경이 성능을 올렸는지 모름 |
| 협업 불일치 | 팀원과 서로 다른 모델로 작업 중 |
| 관리 안 됨 | 모델 파일이 `model_최종.pkl`, `model_진짜최종.pkl` ... |

### MLflow가 해결하는 방법

실험할 때마다 이 모든 것을 **자동으로 기록**한다.

```python
with mlflow.start_run():
    mlflow.log_param("learning_rate", 0.001)   # 설정값 기록
    mlflow.log_metric("accuracy", 0.92)         # 결과 기록
    mlflow.log_model(model, "model")            # 모델 파일 저장
```

그러면 나중에 UI에서 한눈에 비교할 수 있다.

## MLflow의 4가지 핵심 기능

### 1. Tracking (실험 추적)

실험마다 파라미터·메트릭·파일을 자동 기록한다.

```
Run 1: lr=0.01,   accuracy=0.80  ← 기록됨
Run 2: lr=0.001,  accuracy=0.85  ← 기록됨
Run 3: lr=0.001,  accuracy=0.92  ← 기록됨 ✅ 최고 성능
```

### 2. Dataset Tracking (데이터 추적)

어떤 데이터로 학습했는지 함께 기록한다.

```
Run 3: lr=0.001, accuracy=0.92
        ├── 파라미터: learning_rate=0.001
        ├── 메트릭: accuracy=0.92
        └── 데이터: iris-train-v2 (digest: 4a8b...)  ← 어떤 데이터인지
```

### 3. Model Registry (모델 관리)

모델을 버전별로 관리하고, **Alias(별칭)** 로 어느 버전이 어떤 용도인지 표시한다.

```
모델: "iris-classifier"
├── v1               (구버전)
├── v2  @challenger  (테스트 중인 후보)
└── v3  @champion    (현재 서비스 중) ✅
```

> **참고:** 예전 MLflow는 Staging/Production 같은 고정 "Stage"를 썼지만,
> MLflow 2.9부터 deprecated되어 자유롭게 이름 붙이는 **Alias** 방식으로 바뀌었다.

### 4. UI (웹 대시보드)

브라우저에서 실험 결과를 시각적으로 비교한다.

```
http://localhost:5000 접속 → 모든 실험 결과를 표와 그래프로 확인
```

## MLflow 구조 한눈에 보기

```
Experiment (실험 프로젝트)
│   예: "고양이-강아지-분류기"
│
├── Run 1 (1번 실험)
│   ├── 파라미터: epochs=10, lr=0.01
│   ├── 메트릭:   accuracy=0.80
│   ├── 데이터:   train-v1 (digest: abc...)
│   └── 모델:     model.pkl
│
├── Run 2 (2번 실험)
│   ├── 파라미터: epochs=20, lr=0.001
│   ├── 메트릭:   accuracy=0.92
│   ├── 데이터:   train-v2 (digest: def...)
│   └── 모델:     model.pkl
│
└── Run 3 (3번 실험)
    └── ...
```

## MLflow vs 직접 관리

| | 직접 관리 | MLflow |
|-|----------|--------|
| 실험 기록 | 엑셀이나 노트에 직접 적음 | 자동 기록 |
| 모델 저장 | `model_최종2_진짜.pkl` | 버전별 자동 저장 |
| 결과 비교 | 눈으로 하나씩 확인 | UI에서 한눈에 비교 |
| 팀 협업 | 파일을 메신저로 공유 | Tracking Server로 공유 |
| 이전 버전 복원 | 파일 찾아 헤맴 | Run ID로 즉시 복원 |
