# 모델 서빙 (mlflow models serve)

> 모델 저장을 먼저 익혔다면 → `04_모델_관리.md` 복습 후 진행

## 이번 실습 목표

- `mlflow models serve`로 저장된 모델을 API 서버로 실행한다.
- PowerShell에서 API에 데이터를 보내고 예측 결과를 받는다.
- 서빙이 무엇인지, 왜 필요한지 이해한다.

---

## 모델 서빙이란?

학습이 끝난 모델을 **다른 사람·프로그램이 사용할 수 있도록 API로 제공**하는 것이다.

```
[학습 완료된 모델]
        ↓
mlflow models serve
        ↓
[API 서버 실행]  http://localhost:5001/invocations
        ↓
[누구든 HTTP 요청으로 예측 가능]
  요청: {"inputs": [[5.1, 3.5, 1.4, 0.2]]}
  응답: {"predictions": [0]}   ← 예측 결과
```

> **비유:** 학교 급식실처럼, 만들어진 음식(모델)을
> 창구(API)를 통해 누구나(다른 프로그램) 받아갈 수 있게 하는 것이다.

### 왜 서빙이 필요한가?

| 상황 | 서빙 없이 | 서빙 있이 |
|------|----------|----------|
| 웹 서비스에서 모델 사용 | Python 코드를 직접 넣어야 함 | HTTP 요청만 보내면 됨 |
| 다른 언어(JS, Java)에서 사용 | 불가능 | HTTP로 어디서든 가능 |
| 모델 교체 | 전체 코드 수정 필요 | 서버만 재시작 |

## STEP 1. 서빙할 모델 준비

먼저 Registry에 저장된 모델이 있어야 한다.
`04_모델_관리.md`를 진행했다면 이미 있다. 없으면 아래 코드로 만든다.

```python
# prepare_model.py
import mlflow
import mlflow.sklearn
from sklearn.datasets import load_iris
from sklearn.linear_model import LogisticRegression
from sklearn.model_selection import train_test_split

mlflow.set_experiment("serving-demo")

iris = load_iris()
X_train, X_test, y_train, y_test = train_test_split(
    iris.data, iris.target, test_size=0.2, random_state=42
)

with mlflow.start_run(run_name="서빙용-모델") as run:
    model = LogisticRegression(max_iter=200)
    model.fit(X_train, y_train)

    mlflow.sklearn.log_model(
        model,
        artifact_path="model",
        registered_model_name="iris-serving",
        input_example=X_test[:3],
    )

    print(f"Run ID: {run.info.run_id}")
    print("모델 저장 완료!")
```

```powershell
& $python prepare_model.py
# Run ID: a1b2c3d4...
# 모델 저장 완료!
```

Run ID를 메모해두자. 서빙에 사용한다.

## STEP 2. 서빙 전 준비 — PATH 설정 확인

MLflow 3.x는 서빙에 **uvicorn**을 사용한다.
uvicorn 실행 파일이 PATH에 있어야 서버가 뜬다. `01_환경설정.md`의 변수 설정을 먼저 입력하자.

```powershell
$env:PYTHONUTF8 = "1"
$env:PATH = "C:\Users\myesu\miniconda3\Scripts;" + $env:PATH
```

> **MLflow 버전별 서빙 서버 변경:**
> - 2.x Linux/Mac: gunicorn / 2.x Windows: waitress
> - **3.x 전체: uvicorn** (현재 버전)

## STEP 3. 모델 서버 실행

### 방법 1 — Run ID로 서빙

```powershell
$RUN_ID = "a1b2c3d4..."   # prepare_model.py에서 출력된 Run ID

& $mlflow models serve `
    --model-uri "runs:/$RUN_ID/model" `
    --port 5001 `
    --no-conda
```

### 방법 2 — Model Registry로 서빙 (추천)

```powershell
# 버전 번호로
& $mlflow models serve `
    --model-uri "models:/iris-serving/1" `
    --port 5001 `
    --no-conda

# Alias로 (champion 버전을 자동 선택)
& $mlflow models serve `
    --model-uri "models:/iris-serving@champion" `
    --port 5001 `
    --no-conda
```

### 서버 시작 확인

아래 메시지가 나오면 서버가 정상 실행된 것이다.

```
INFO:     Uvicorn running on http://127.0.0.1:5001 (Press CTRL+C to quit)
```

> **`--no-conda` 옵션이란?**
> conda 가상환경을 새로 만들지 않고 현재 환경을 그대로 사용한다.
> 실습 환경에서는 이 옵션이 필요하다.

> **포트 번호를 5001로 쓰는 이유:**
> MLflow UI가 이미 5000번을 사용 중이기 때문에 5001번을 사용한다.
> `--port` 뒤에 원하는 번호를 쓰면 된다.

## STEP 4. API에 예측 요청 보내기

서버가 실행 중인 상태에서 **새 터미널**을 열고 요청을 보낸다.

### 요청 데이터 형식

```json
{
  "inputs": [[5.1, 3.5, 1.4, 0.2]]
}
```

> 붓꽃 데이터의 4가지 특징값: 꽃받침 길이, 꽃받침 너비, 꽃잎 길이, 꽃잎 너비

### PowerShell로 요청

```powershell
# 붓꽃 1개 예측
$body = '{"inputs": [[5.1, 3.5, 1.4, 0.2]]}'

Invoke-RestMethod `
    -Uri "http://localhost:5001/invocations" `
    -Method POST `
    -ContentType "application/json" `
    -Body $body
```

응답 결과:

```json
{
  "predictions": [0]
}
```

> **0이 무슨 뜻?**
> 붓꽃 3종류를 숫자로 표현한 것이다.
> 0 = Setosa, 1 = Versicolor, 2 = Virginica

### 여러 개 한번에 예측

```powershell
$body = '{"inputs": [[5.1, 3.5, 1.4, 0.2], [6.7, 3.0, 5.2, 2.3], [5.8, 2.7, 4.1, 1.0]]}'

Invoke-RestMethod `
    -Uri "http://localhost:5001/invocations" `
    -Method POST `
    -ContentType "application/json" `
    -Body $body
```

응답:

```json
{
  "predictions": [0, 2, 1]
}
```

## STEP 5. Python 코드에서 API 호출하기

실제 서비스에서는 Python 코드로 API를 호출한다.

```python
# call_api.py
import requests
import json

API_URL = "http://localhost:5001/invocations"

# 예측할 데이터
data = {
    "inputs": [
        [5.1, 3.5, 1.4, 0.2],   # Setosa 예상
        [6.7, 3.0, 5.2, 2.3],   # Virginica 예상
        [5.8, 2.7, 4.1, 1.0],   # Versicolor 예상
    ]
}

response = requests.post(
    url=API_URL,
    headers={"Content-Type": "application/json"},
    data=json.dumps(data),
)

result = response.json()
predictions = result["predictions"]

# 숫자 → 품종 이름으로 변환
species = {0: "Setosa", 1: "Versicolor", 2: "Virginica"}

print("=== 예측 결과 ===")
for i, pred in enumerate(predictions):
    print(f"샘플 {i+1}: {species[pred]} (클래스 {pred})")
```

```powershell
# requests 설치 (없다면)
& "C:\Users\myesu\miniconda3\Scripts\pip.exe" install requests

& $python call_api.py
# === 예측 결과 ===
# 샘플 1: Setosa (클래스 0)
# 샘플 2: Virginica (클래스 2)
# 샘플 3: Versicolor (클래스 1)
```

## STEP 7. 모델 교체하기 (무중단)

Registry의 스테이지를 바꾸면 서버 재시작만으로 다른 버전을 서빙할 수 있다.

```python
# switch_model.py
import mlflow

client = mlflow.tracking.MlflowClient()

# Version 2에 champion alias 부여 (기존 Version 1의 champion을 교체)
client.set_registered_model_alias(
    name="iris-serving",
    alias="champion",
    version=2,
)
print("Version 2가 champion으로 변경됨")
```

```powershell
& $python switch_model.py

# 서버 재시작
# Ctrl+C 로 서버 종료 후
& $mlflow models serve --model-uri "models:/iris-serving@champion" --port 5001 --no-conda
# 이제 Version 2가 응답함
```

## 전체 서빙 흐름 정리

```
[1. 모델 학습]
mlflow.sklearn.log_model(..., registered_model_name="iris-serving")

[2. Alias 설정]
client.set_registered_model_alias(name="iris-serving", alias="champion", version=1)

[3. 서버 실행]
mlflow models serve --model-uri "models:/iris-serving@champion" --port 5001 --no-conda

[4. 예측 요청]
POST http://localhost:5001/invocations
{"inputs": [[5.1, 3.5, 1.4, 0.2]]}

[5. 예측 응답]
{"predictions": [0]}
```

## API 엔드포인트 정리

| URL | 메서드 | 설명 |
|-----|--------|------|
| `/invocations` | POST | 예측 요청 (핵심) |
| `/health` | GET | 서버 상태 확인 |
| `/version` | GET | MLflow 버전 확인 |

---

## 요청 데이터 형식

MLflow 서버는 두 가지 형식을 모두 받는다.

```json
// 형식 1: inputs (컬럼 순서 기준)
{"inputs": [[5.1, 3.5, 1.4, 0.2]]}

// 형식 2: dataframe_records (컬럼 이름 기준)
{"dataframe_records": [
    {"sepal length (cm)": 5.1, "sepal width (cm)": 3.5,
     "petal length (cm)": 1.4, "petal width (cm)": 0.2}
]}
```

## 자주 하는 실수

| 실수 | 원인 | 해결 |
|------|------|------|
| 서버 시작 오류 | conda 환경 문제 | `--no-conda` 옵션 추가 |
| 포트 충돌 | 5001번 이미 사용 중 | `--port 5002` 등으로 변경 |
| 예측 요청 오류 | 입력 데이터 형식 틀림 | `"inputs"` 키 확인, 2차원 배열로 감싸기 |
| 서버가 꺼짐 | 터미널을 닫아버림 | 서버 전용 터미널을 따로 유지 |
| 모델 못 찾음 | Run ID 또는 버전 오타 | UI에서 복사해서 붙여넣기 |
