# 모델 서빙 (mlflow models serve)

> 모델 저장을 먼저 익혔다면 → `04_모델_관리.md` 복습 후 진행

## 이번 실습 목표

- `mlflow models serve`로 저장된 모델을 API 서버로 실행한다.
- Python 코드로 API에 데이터를 보내고 예측 결과를 받는다.
- 서빙이 무엇인지, 왜 필요한지 이해한다.

---

## 모델 서빙이란?

학습이 끝난 모델을 **다른 사람·프로그램이 사용할 수 있도록 API로 제공**하는 것이다.

```
[학습 완료된 모델]
        ↓
mlflow models serve
        ↓
[API 서버 실행]  http://localhost:5001/invocations
        ↓
[누구든 HTTP 요청으로 예측 가능]
  요청: {"inputs": [[5.1, 3.5, 1.4, 0.2]]}
  응답: {"predictions": [0]}
```

> **비유:** 학교 급식실처럼, 만들어진 음식(모델)을
> 창구(API)를 통해 누구나(다른 프로그램) 받아갈 수 있게 하는 것이다.

### 왜 서빙이 필요한가?

| 상황 | 서빙 없이 | 서빙 있이 |
|------|----------|----------|
| 웹 서비스에서 모델 사용 | Python 코드를 직접 넣어야 함 | HTTP 요청만 보내면 됨 |
| 다른 언어(JS, Java)에서 사용 | 불가능 | HTTP로 어디서든 가능 |
| 모델 교체 | 전체 코드 수정 필요 | 서버 재시작만 하면 됨 |

## STEP 1. 서버 실행 — `01_start_server.py`

```python
# [07-1] MLflow 모델 서버 실행
# 목표: iris-classifier@champion 모델을 API 서버로 백그라운드 실행한다
# 핵심: 서버는 이 스크립트가 끝난 뒤에도 계속 실행됨
# 다음 파일: 02_call_api.py

import os, sys, time, subprocess, requests

MLFLOW   = r"C:\Users\myesu\miniconda3\Scripts\mlflow.exe"
DB_URI   = "sqlite:///C:/Users/myesu/Project/MLOps_Study/MLflow/practice/mlflow.db"
PORT     = 5001
PID_FILE = "server.pid"

# ① uvicorn이 있는 Scripts 폴더를 PATH에 추가
#    MLFLOW_TRACKING_URI로 DB 경로 전달
#    (mlflow models serve 에는 --backend-store-uri 옵션이 없음)
env = os.environ.copy()
env["PATH"]                = r"C:\Users\myesu\miniconda3\Scripts;" + env.get("PATH", "")
env["PYTHONUTF8"]          = "1"
env["MLFLOW_TRACKING_URI"] = DB_URI

# ② 이미 실행 중인지 확인
try:
    r = requests.get(f"http://localhost:{PORT}/health", timeout=1)
    if r.status_code == 200:
        print(f"서버가 이미 실행 중입니다. (port {PORT})")
        sys.exit(0)
except Exception:
    pass

# ③ 서버 백그라운드 실행
#    server.log 에 서버 출력 저장 → 문제 발생 시 확인 가능
log_file = open("server.log", "w")
proc = subprocess.Popen(
    [MLFLOW, "models", "serve",
     "--model-uri", "models:/iris-classifier@champion",
     "--port",      str(PORT),
     "--no-conda"],
    env=env, stdout=log_file, stderr=log_file,
)

# ④ PID 저장 — 03_stop_server.py 에서 종료할 때 사용
with open(PID_FILE, "w") as f:
    f.write(str(proc.pid))

print(f"서버 시작 중... (PID: {proc.pid})")

# ⑤ 서버가 준비될 때까지 대기 (최대 30초)
for i in range(30):
    try:
        r = requests.get(f"http://localhost:{PORT}/health", timeout=1)
        if r.status_code == 200:
            print(f"\n서버 준비 완료! → http://localhost:{PORT}")
            print("02_call_api.py 를 실행하세요.")
            sys.exit(0)
    except Exception:
        print(".", end="", flush=True)
        time.sleep(1)

print("\n서버 시작 시간 초과. 로그를 확인하세요.")
proc.terminate()
sys.exit(1)
```

```powershell
& $python 01_start_server.py
# 서버 시작 중... (PID: 12345)
# ..............
# 서버 준비 완료! → http://localhost:5001
# 02_call_api.py 를 실행하세요.
```

> **서버 시작이 오래 걸릴 때:** `server.log` 에 서버 출력이 저장된다.
> ```powershell
> Get-Content server.log -Wait
> ```

## STEP 2. 예측 요청 — `02_call_api.py`

```python
# [07-2] 서빙 중인 모델에 예측 요청
# 목표: 실행 중인 API 서버에 HTTP 요청을 보내 예측값을 받는다
# 실행 전: 01_start_server.py 실행 후 서버가 준비되어야 함
# 다음 파일: 03_stop_server.py

import json, requests

API_URL = "http://localhost:5001/invocations"

# ① 예측할 샘플 데이터 3개
#    [꽃받침 길이, 꽃받침 너비, 꽃잎 길이, 꽃잎 너비]
data = {
    "inputs": [
        [5.1, 3.5, 1.4, 0.2],   # Setosa 예상
        [6.7, 3.0, 5.2, 2.3],   # Virginica 예상
        [5.8, 2.7, 4.1, 1.0],   # Versicolor 예상
    ]
}

# ② API 요청
try:
    response = requests.post(
        url=API_URL,
        headers={"Content-Type": "application/json"},
        data=json.dumps(data),
    )
    response.raise_for_status()

    # ③ 결과 출력
    predictions = response.json()["predictions"]
    species = {0: "Setosa", 1: "Versicolor", 2: "Virginica"}

    print("=== 예측 결과 ===")
    for i, pred in enumerate(predictions):
        print(f"샘플 {i+1}: {species[pred]} (클래스 {pred})")

except requests.exceptions.ConnectionError:
    print("[오류] 서버에 연결할 수 없습니다.")
    print("01_start_server.py 를 먼저 실행하세요.")
```

```powershell
& $python 02_call_api.py
# === 예측 결과 ===
# 샘플 1: Setosa (클래스 0)
# 샘플 2: Virginica (클래스 2)
# 샘플 3: Versicolor (클래스 1)
```

### 요청 데이터 형식

MLflow 서버는 두 가지 형식을 모두 받는다.

```json
// 형식 1: inputs (컬럼 순서 기준) — 권장
{"inputs": [[5.1, 3.5, 1.4, 0.2]]}

// 형식 2: dataframe_records (컬럼 이름 기준)
{"dataframe_records": [
    {"sepal length (cm)": 5.1, "sepal width (cm)": 3.5,
     "petal length (cm)": 1.4, "petal width (cm)": 0.2}
]}
```

## STEP 3. 서버 종료 — `03_stop_server.py`

```python
# [07-3] 모델 서버 종료
# 목표: 01_start_server.py 가 실행한 서버 프로세스를 종료한다
# 실행 전: 01_start_server.py 실행 후 server.pid 파일이 있어야 함

import os, sys, signal

PID_FILE = "server.pid"

# ① PID 파일 확인
if not os.path.exists(PID_FILE):
    print("[오류] server.pid 파일이 없습니다. 01_start_server.py 로 서버를 시작하세요.")
    sys.exit(1)

# ② PID 읽기
with open(PID_FILE) as f:
    pid = int(f.read().strip())

# ③ 프로세스 종료
try:
    os.kill(pid, signal.SIGTERM)
    os.remove(PID_FILE)
    print(f"서버 종료 완료. (PID: {pid})")
except ProcessLookupError:
    print(f"PID {pid} 프로세스가 이미 종료되어 있습니다.")
    os.remove(PID_FILE)
```

```powershell
& $python 03_stop_server.py
# 서버 종료 완료. (PID: 12345)
```

## 모델 교체하기

Registry의 Alias만 바꾸면 서버 재시작만으로 다른 버전을 서빙할 수 있다.

```python
import mlflow

mlflow.set_tracking_uri("sqlite:///C:/Users/myesu/Project/MLOps_Study/MLflow/practice/mlflow.db")

client = mlflow.tracking.MlflowClient()
client.set_registered_model_alias(name="iris-classifier", alias="champion", version=2)
print("Version 2가 champion으로 변경됨")
```

```powershell
& $python 03_stop_server.py   # 서버 종료
& $python 01_start_server.py  # 재시작 → 이제 Version 2가 응답
```

## 전체 흐름 정리

```
[1. 모델 학습 + Registry 등록]  ← 04_모델_관리.md 에서 완료
[2. Alias 설정]                 ← iris-classifier@champion 이 있어야 함
[3. 서버 실행]   01_start_server.py
[4. 예측 요청]   02_call_api.py  →  POST /invocations
[5. 서버 종료]   03_stop_server.py
```

## API 엔드포인트 정리

| URL | 메서드 | 설명 |
|-----|--------|------|
| `/invocations` | POST | 예측 요청 (핵심) |
| `/health` | GET | 서버 상태 확인 |
| `/version` | GET | MLflow 버전 확인 |

---

## 자주 하는 실수

| 실수 | 원인 | 해결 |
|------|------|------|
| `No such option: --backend-store-uri` | serve 명령에 없는 옵션 | `MLFLOW_TRACKING_URI` 환경변수로 전달 |
| 서버 시작 오류 | uvicorn PATH 미설정 | Scripts 폴더를 PATH에 추가 |
| 포트 충돌 | 5001번 이미 사용 중 | `PORT = 5002` 로 변경 후 재실행 |
| 예측 요청 오류 | 입력 데이터 형식 틀림 | `"inputs"` 키 확인, 2차원 배열로 감싸기 |
| 모델 못 찾음 | `@champion` alias 미설정 | `04_모델_관리.md` → `02_set_alias.py` 먼저 실행 |
| 서버가 응답 없음 | 시작 중에 요청 보냄 | `01_start_server.py` 가 "준비 완료" 출력 후 요청 |
