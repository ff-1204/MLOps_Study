## 2. MLflow 설치

```powershell
& "C:\Users\myesu\miniconda3\Scripts\pip.exe" install mlflow scikit-learn pandas
```

> **왜 scikit-learn과 pandas도 같이 설치하나?**
> 실습에서 간단한 ML 모델(scikit-learn)과 데이터 처리(pandas)를 사용하기 때문이다.

### 설치 확인

```powershell
& "C:\Users\myesu\miniconda3\python.exe" -c "import mlflow; print(mlflow.__version__)"
# 3.13.0
```

### 편의를 위한 변수 저장

매번 긴 경로를 입력하기 불편하니 PowerShell 변수로 저장해두자.
아래 3줄을 **터미널을 열 때마다** 입력한다.

```powershell
$python = "C:\Users\myesu\miniconda3\python.exe"
$env:PYTHONUTF8 = "1"
$env:PATH = "C:\Users\myesu\miniconda3\Scripts;" + $env:PATH
```

> `PYTHONUTF8=1` — MLflow 자동 로깅(`autolog`)이 Windows CP949 인코딩 오류 없이 동작하게 한다.
> `PATH` 설정 — `mlflow models serve` 내부에서 uvicorn을 찾지 못하는 오류를 방지한다.
> **주의:** 터미널을 닫고 새로 열면 초기화된다. 다시 입력해야 한다.

## 3. MLflow UI 실행

실습 폴더에 `practice/01/start_mlflow.py` 스크립트가 준비되어 있다.

```powershell
& "C:\Users\myesu\miniconda3\python.exe" "C:\Users\myesu\Project\MLOps_Study\MLflow\practice\01\start_mlflow.py"
```

아래 메시지가 나오면 정상이다.

```
MLflow UI 시작 중...
브라우저에서 http://localhost:5000 접속
종료하려면 Ctrl+C
INFO:     Uvicorn running on http://127.0.0.1:5000 (Press CTRL+C to quit)
```

브라우저에서 `http://localhost:5000` 접속

> **포트 충돌 오류가 나면?**
> 5000번 포트를 다른 프로그램이 쓰고 있는 것이다.
> `start_mlflow.py` 마지막 줄에서 `--port 5000`을 `--port 5001`로 바꾸고 `http://localhost:5001`로 접속한다.

> **첫 실행 시** `mlflow.db` 파일이 `practice/01/` 폴더에 자동 생성된다.
> 이 파일이 실험 데이터를 저장하는 SQLite 데이터베이스다.

UI를 닫으려면 터미널에서 `Ctrl + C`를 누른다.
