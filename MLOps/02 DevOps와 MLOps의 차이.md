# DevOps와 MLOps의 차이

---

## DevOps란?

전통적인 SW 개발은 개발팀과 운영팀이 따로 움직였다.

```
개발팀: 코드 짜고 배포 요청
운영팀: 서버에 올리고 관리
```

이 둘이 소통이 안 되다 보니 배포가 느리고 오류 대응도 늦었다.  
이를 해결하기 위해 **DevOps**가 등장했다.

**DevOps** = 개발(Development) + 운영(Operations)을 하나의 팀·프로세스로 통합하는 방식

```
plan → code → build → test → release → deploy → operate → monitor → (반복)
```

핵심은 **무한 반복(Infinite Loop)** 구조다. 배포 이후에도 모니터링하고, 문제가 생기면 바로 코드를 수정해 다시 배포한다.

---

## MLOps란?

ML(머신러닝) 프로젝트에 DevOps 원칙을 적용한 것이다.

```
MLOps = ML + DevOps
```

ML 모델을 학습하는 것에서 끝내지 않고, **서비스로 배포하고 지속적으로 운영·개선**하는 전체 과정을 자동화·체계화한다.

---

## DevOps vs MLOps 핵심 차이

### 1. 관리 대상이 다르다

DevOps는 **코드**만 관리하면 됐다.  
MLOps는 **코드 + 데이터 + 모델** 세 가지를 모두 관리해야 한다.

```
AI Service = SW(코드) + Data(데이터)
```

| 관리 대상 | DevOps | MLOps |
|---|---|---|
| 코드 | ✅ | ✅ |
| 데이터 | ❌ | ✅ |
| 모델 | ❌ | ✅ |

> 같은 코드라도 **다른 데이터로 학습하면 다른 모델**이 나온다.  
> 그래서 MLOps에서는 "어떤 코드로 어떤 데이터를 써서 어떤 모델을 만들었는가"를 모두 기록해야 한다.

---

### 2. 테스트 방식이 다르다

| 구분 | DevOps | MLOps |
|---|---|---|
| 테스트 대상 | 코드가 올바르게 동작하는가 | 모델 성능이 기준치 이상인가 |
| 테스트 결과 | 성공 / 실패 (명확함) | 정확도 92% / 93% (상대적 판단 필요) |
| 테스트 도구 | 유닛 테스트, 통합 테스트 | 모델 평가 지표 (Accuracy, F1-Score 등) |

> SW 테스트는 "버튼을 누르면 결과가 나오는가" 같은 명확한 기준이 있다.  
> ML 테스트는 "이 모델이 충분히 좋은가"를 사람이 판단해야 한다는 차이가 있다.

---

### 3. 배포 방식이 다르다

| 구분 | DevOps | MLOps |
|---|---|---|
| 배포 대상 | 새 버전의 코드 | 새 버전의 모델 파일 |
| 배포 트리거 | 코드 변경(git push) | 코드 변경 또는 **새 데이터 축적** |
| 롤백 기준 | 오류 발생 시 이전 코드로 | 성능 저하 감지 시 이전 모델로 |

> MLOps에서는 코드가 바뀌지 않아도 **데이터가 충분히 쌓이면 자동으로 재학습·재배포**하는 파이프라인을 만든다.

---

### 4. 모니터링 대상이 다르다

DevOps는 서버가 살아있는지, 응답 속도는 빠른지 감시한다.  
MLOps는 거기에 더해 **모델 성능과 데이터 변화**까지 감시해야 한다.

| 모니터링 항목 | DevOps | MLOps |
|---|---|---|
| 서버 상태 (CPU, 메모리) | ✅ | ✅ |
| API 응답 속도 | ✅ | ✅ |
| 모델 정확도 변화 | ❌ | ✅ |
| 데이터 드리프트 | ❌ | ✅ |

> **데이터 드리프트(Data Drift)**: 코드와 모델은 그대로인데 입력 데이터의 패턴이 달라져서 성능이 떨어지는 현상.  
> 예) 작년 구매 데이터로 학습한 쇼핑몰 추천 모델이, 올해 소비 트렌드가 바뀌면서 엉뚱한 상품을 추천하게 되는 경우.

---

### 5. 실험 관리가 추가된다

DevOps에는 없는 개념이다.

ML 개발에서는 수십~수백 번의 실험을 반복한다.

```
실험 1: learning_rate=0.001, epochs=10  →  정확도 87%
실험 2: learning_rate=0.01,  epochs=20  →  정확도 91%
실험 3: learning_rate=0.001, epochs=30  →  정확도 93%  ← 채택
```

이 과정을 수동으로 기록하면 나중에 재현이 불가능하다.  
MLOps는 **실험 추적 도구(MLflow 등)** 로 모든 실험의 파라미터와 결과를 자동으로 기록한다.

---

## 전체 비교 요약

| 항목 | DevOps | MLOps |
|---|---|---|
| 관리 대상 | 코드 | 코드 + 데이터 + 모델 |
| 버전 관리 | 코드 버전 (Git) | 코드 + 데이터 + 모델 버전 |
| 테스트 | 코드 동작 검증 | 모델 성능 평가 |
| 배포 트리거 | 코드 변경 | 코드 변경 또는 데이터 축적 |
| 모니터링 | 서버·서비스 상태 | 서버 상태 + 모델 성능 + 데이터 드리프트 |
| 추가 개념 | 없음 | 실험 관리, 피처 스토어, 모델 레지스트리 |

---

## 정리

DevOps가 "SW 개발과 운영의 통합"이라면,  
MLOps는 "ML 개발·실험·배포·운영의 통합"이다.

DevOps의 원칙(자동화, 빠른 피드백, 지속적 배포)을 그대로 가져오되,  
ML 특유의 문제(데이터 관리, 실험 추적, 모델 드리프트)를 추가로 해결한다.
